# Defining Tables

Every class that maps to a database table must be decorated with `[Table]` and declared `partial` so the source generator can augment it.

## `[Table]`

```csharp
[Table("users")]
public partial class User { ... }
```

The argument is the exact SQL table name. The generator uses this name everywhere — migration DDL, INSERT/SELECT statements, generated extension methods.

### Enums as reference tables

Plain (non-flagged) enums can carry `[Table]` to represent a reference/lookup table. The generated table always has `id`, `value`, and `description` columns. Use `[Description]` on enum members to populate the `description` column:

```csharp
[Table("user_visibility")]
public enum UserVisibility : short
{
    [Description("Visible to everyone")]
    Public,

    [Description("Visible to circles only")]
    CirclesOnly,

    CustomCircles   // description column will be empty
}
```

The migration tool seeds the table with all enum values on the first migration.

## `[PrimaryKey]`

```csharp
[PrimaryKey]
public Guid Id { get; set; }
```

Marks the property as the primary key. Composite PKs are supported — apply `[PrimaryKey]` to every participating column:

```csharp
[PrimaryKey, ForeignKey(typeof(User))]
public Guid UserId { get; set; }

[PrimaryKey, ForeignKey(typeof(Course))]
public Guid CourseId { get; set; }
```

## `[Default]`

Declares a DB-side default value. The insert builder's `ExcludeAutoFields()` skips columns marked `[Default]` so the database applies the default.

```csharp
// Random UUID generated by PostgreSQL
[Default(DbDefaults.Guid.Random)]
public Guid Id { get; set; }

// Current UTC timestamp at insert time
[Default(DbDefaults.Time.Now)]
public DateTime CreatedAt { get; set; }

// Explicit SQL expression
[Default("'pending'")]
public string Status { get; set; } = "";

// Naked DEFAULT — just skipped by ExcludeAutoFields, no expression emitted
[Default]
public Guid Id { get; set; }
```

See [DbDefaults & DbValues](db-constants.md) for the full list of built-in constants.

### C# property initializers as defaults

If a property has no `[Default]` attribute, the migration tool reads the C# initializer value and uses it as the SQL `DEFAULT`:

```csharp
public string Name { get; set; } = "DEFAULT NAME";
// → "name" TEXT NOT NULL DEFAULT 'DEFAULT NAME'

public int Priority { get; set; } = 5;
// → "priority" INTEGER NOT NULL DEFAULT 5
```

`[Default]` always takes precedence over a C# initializer. Initializers equal to the type's zero value (`null`, `0`, `false`, `Guid.Empty`) are ignored.

## `[Column]`

Overrides the auto-derived column name or SQL type:

```csharp
// Custom column name
[Column("email_address")]
public string Email { get; set; } = "";

// Custom SQL type
[Column(Type = "JSONB")]
public string Metadata { get; set; } = "";

// Both
[Column("meta", Type = "JSONB")]
public string Metadata { get; set; } = "";
```

Without `[Column]`, the property name is converted to `snake_case` automatically.

## C# to PostgreSQL type mapping

The migration tool infers the SQL column type from the C# property type. The full mapping is:

| C# type | PostgreSQL type |
|---------|----------------|
| `string` | `text` |
| `char` | `character(1)` |
| `bool` / `boolean` | `boolean` |
| `short` / `int16` | `smallint` |
| `int` / `int32` | `integer` |
| `long` / `int64` | `bigint` |
| `float` / `single` | `real` |
| `double` | `double precision` |
| `decimal` | `numeric` |
| `Guid` | `uuid` |
| `DateTime` | `timestamp without time zone` |
| `DateTimeOffset` | `timestamp with time zone` |
| `DateOnly` | `date` |
| `TimeOnly` | `time without time zone` |
| `TimeSpan` | `interval` |
| `byte[]` | `bytea` |

When the inferred type is not what you need, override it with `[Column(Type = "...")]`:

```csharp
// Force a specific precision
[Column(Type = "NUMERIC(10,2)")]
public decimal Price { get; set; }

// Store as date (DateOnly is inferred automatically, but explicit override also works)
[Column(Type = "DATE")]
public DateOnly BirthDate { get; set; }
```

### `DateOnly` and `TimeOnly`

`System.DateOnly` maps to `date` and `System.TimeOnly` maps to `time without time zone`. They work in every context — migrations, INSERT, UPDATE, WHERE clauses, and reads:

```csharp
[Table("events")]
public partial class Event
{
    [PrimaryKey, Default(DbDefaults.Guid.Random)]
    public Guid Id { get; set; }

    public string Name { get; set; } = "";

    // Stored as PostgreSQL DATE
    public DateOnly Date { get; set; }

    // Stored as PostgreSQL TIME WITHOUT TIME ZONE
    public TimeOnly StartTime { get; set; }

    // Nullable variants work too
    public TimeOnly? EndTime { get; set; }
}
```

```csharp
// Insert
var ev = new Event
{
    Name     = "Team standup",
    Date     = new DateOnly(2026, 6, 1),
    StartTime = new TimeOnly(9, 30),
};
await ev.Insert().WithConnection(conn).ExcludeAutoFields().ExecuteAsync();

// Query with WHERE
await foreach (var e in Event.Query(x => x.Date == new DateOnly(2026, 6, 1))
    .WithConnection(conn).ExecuteAsync())
{
    Console.WriteLine($"{e.Name} at {e.StartTime}");
}
```

> **Requires Npgsql 6 or later.** Earlier versions map `date` columns to `DateTime`; the `DateOnly`/`TimeOnly` ADO.NET types were added in Npgsql 6.

## `[Nullable]`

Marks a column as nullable (`NULL` instead of `NOT NULL`):

```csharp
[Nullable]
public string? IconUrl { get; set; }
```

Nullable reference types (`string?`) are detected automatically, so `[Nullable]` is mainly needed for value types:

```csharp
[Nullable]
public int? OptionalScore { get; set; }
```

## `[Ignore]`

Excludes a property entirely from the generated schema and all SQL:

```csharp
[Ignore]
public string FullName => $"{FirstName} {LastName}";
```

## `[ForeignKey]`

### Property-level (single column)

```csharp
[ForeignKey(typeof(User))]
public Guid UserId { get; set; }
```

> **Warning:** The short form `[ForeignKey(typeof(User))]` only works when the target table has **exactly one primary key** and its type matches the property type. If `User` has a composite PK or a mismatched type, specify the target key explicitly using `nameof()`:
>
> ```csharp
> [ForeignKey(typeof(User), TargetKeys = new[] { nameof(User.Id) })]
> public Guid UserId { get; set; }
> ```

### Referential actions

Use `DbValues.ForeignKey.*` constants so your code stays portable across database engines:

```csharp
[ForeignKey(typeof(User), OnDelete = DbValues.ForeignKey.Cascade)]
public Guid UserId { get; set; }

[ForeignKey(typeof(User),
    OnDelete = DbValues.ForeignKey.SetNull,
    OnUpdate = DbValues.ForeignKey.NoAction)]
public Guid? AuthorId { get; set; }
```

Raw SQL strings (`OnDelete = "CASCADE"`) also work but are not portable.

Available constants: `Cascade`, `SetNull`, `SetDefault`, `Restrict`, `NoAction`.

### Class-level (composite or cross-column)

```csharp
[Table("user_course_agreement")]
[ForeignKey(typeof(UserCourse),
    Keys = new[] { nameof(UserId), nameof(CourseId) },
    TargetKeys = new[] { nameof(UserCourse.UserId), nameof(UserCourse.CourseId) },
    OnDelete = DbValues.ForeignKey.Cascade)]
public partial class UserCourseAgreement
{
    public Guid UserId { get; set; }
    public Guid CourseId { get; set; }
}
```

`Name` — optional constraint name:

```csharp
[ForeignKey(typeof(User), Name = "fk_post_author", OnDelete = DbValues.ForeignKey.Cascade)]
public Guid AuthorId { get; set; }
```

## `[Unique]`

### Property-level (single column)

```csharp
[Unique]
public string Email { get; set; } = "";
```

### Class-level (multi-column)

```csharp
[Unique(Columns = new[] { nameof(FirstName), nameof(LastName) }, Name = "uq_full_name")]
public partial class Person { ... }
```

## `[Renamed]`

Tells the migration tool that a column was renamed (instead of drop + add):

```csharp
[Renamed("old_column_name")]
public string NewColumnName { get; set; } = "";
```

## `[RawJsonColumn]` / `[JsonColumn]`

Mark a property as a `jsonb` PostgreSQL column. Use `[RawJsonColumn]` for plain string storage, or `[JsonColumn(typeof(TContext))]` for AOT-safe typed serialization backed by a `JsonSerializerContext`.

```csharp
[RawJsonColumn]
public string? Metadata { get; set; }

[JsonColumn(typeof(MyDbJsonContext))]
public UserProfile? Profile { get; set; }
```

See [JSON Columns](json-columns.md) for setup steps, serialization details, and migration behaviour.

## `[DatabaseEnum]`

Marks an enum so the generator creates a PostgreSQL `ENUM` type instead of a plain integer column.

## Full example

```csharp
[Table("posts")]
[Unique(Columns = new[] { nameof(AuthorId), nameof(Slug) }, Name = "uq_post_slug")]
public partial class Post
{
    [PrimaryKey, Default(DbDefaults.Guid.Random)]
    public Guid Id { get; set; }

    [ForeignKey(typeof(User), OnDelete = DbValues.ForeignKey.Cascade)]
    public Guid AuthorId { get; set; }

    [StringLength(200, MinLength = 1)]
    public string Title { get; set; } = "";

    [Column("slug"), Unique]
    public string Slug { get; set; } = "";

    public string Body { get; set; } = "";

    [Default(DbDefaults.Time.Now)]
    public DateTime PublishedAt { get; set; }

    [Nullable]
    public DateTime? DeletedAt { get; set; }
}
```
